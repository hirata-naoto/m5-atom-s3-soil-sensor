# SEN0605 Modbus Registers

DFRobot RS485 MODBUS-RTU Soil NPK Measure Sensor (SKU: SEN0605) のModbusレジスタ仕様です。

## 通信パラメータ (Default)

| 項目 | 設定値 | 備考 |
| :--- | :--- | :--- |
| インタフェース | RS-485 | |
| 通信プロトコル | Modbus-RTU | |
| スレーブアドレス | `0x01` (1) | `0x01`〜`0xF7` (1〜247) まで変更可能 |
| ボーレート | 9600 bps | 2400 / 4800 / 9600 bps 設定可能 |
| データビット | 8 bit | |
| パリティビット | None | |
| ストップビット | 1 bit | |
| 誤り検出 | CRC-16 (Modbus) | 低バイト、高バイトの順 |

## 保持レジスタ (Holding Registers)

ファンクションコード `0x03` (Read Holding Registers) を使用してデータを読み出します。

### 測定データ

| アドレス (Hex) | アドレス (Dec) | 項目 | 単位 | データ型 | 倍率 | 状態 | 備考 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| `0x001E` | 40031 | 窒素 (N) | mg/kg | 16-bit 符号なし整数 (U16) | 1 | R | 土壌中の窒素含有量 |
| `0x001F` | 40032 | リン (P) | mg/kg | 16-bit 符号なし整数 (U16) | 1 | R | 土壌中のリン含有量 |
| `0x0020` | 40033 | カリウム (K) | mg/kg | 16-bit 符号なし整数 (U16) | 1 | R | 土壌中のカリウム含有量 |

> ⚠️ **注意**: SEN0605は、製品の仕様およびメーカーのファームウェア世代によって、測定データのレジスタアドレスが `0x0000`〜`0x0002`（または `0x0004`〜`0x0006`）に割り当てられているバリエーションが存在します。標準の `0x001E` で値が取得できない（`0` が返る）場合は、上記のアドレスを試してください。

### デバイス設定レジスタ

ファンクションコード `0x06` (Write Single Register) を使用して設定を変更します。

| アドレス (Hex) | アドレス (Dec) | 項目 | データ型 | 設定範囲 / 意味 | 状態 | デフォルト値 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| `0x07d0` | 42001 | スレーブアドレス | U16 | `0x0001` 〜 `0x00F7` (1〜247) | R/W | `0x0001` |
| `0x07d1` | 42002 | ボーレート | U16 | `0`: 2400 bps<br>`1`: 4800 bps<br>`2`: 9600 bps | R/W | `2` (9600) |

## 通信フレーム例

### NPK（窒素・リン・カリウム）の全一括読み出し

**リクエスト (Master -> Sensor):**

```
01 03 00 1E 00 03 65 CD
```
* `01`: スレーブアドレス
* `03`: ファンクションコード (Read)
* `00 1E`: 開始レジスタアドレス (`0x001E` = 窒素)
* `00 03`: 読み出すレジスタ数 (3レジスタ)
* `65 CD`: CRC-16 チェックサム

**レスポンス (Sensor -> Master):**

```
01 03 06 00 20 00 25 00 30 49 92
```
* `01`: スレーブアドレス
* `03`: ファンクションコード
* `06`: データバイト数 (3レジスタ × 2バイト = 6バイト)
* `00 20`: 窒素データ (`0x0020` = 32 mg/kg)
* `00 25`: リンデータ (`0x0025` = 37 mg/kg)
* `00 30`: カリウムデータ (`0x0030` = 48 mg/kg)
* `49 92`: CRC-16 チェックサム

```
### 補足とアドバイス
SEN0605をはじめとするこのタイプの中国系工業用RS485 NPKセンサーは、**外見が全く同じでも内部のチップやOEM元によってレジスタマップが微妙に異なるケース**が多々あります（アドレスが 0x001E のものと、0x0000 から始まるものが混在しています）。
もし元のプログラムでうまく動かない場合は、まずはM5 Atom S3側から 0x001E でリクエストを送り、もしエラーや 0 しか返ってこないようであれば、リクエストの開始アドレスを 0x0000 に書き換えて試してみてください。
